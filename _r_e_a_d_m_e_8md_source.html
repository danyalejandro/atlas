3 ## What is Atlas?
4 It is a simple, light-weight graphics framework designed to support all types of OpenGL applications. Atlas is designed to be a teaching, prototyping, and research framework that allows users to quickly focus on solving graphics problems without having to worry about the details of how to set up things on their own machines. To this end, Atlas provides support for the following things:
5 
6 * Cross-platform creation of windows and OpenGL contexts.
7 * Full vector, matrix, and quaternion library with support for all common graphics operations.
8 * Templated floating point comparison operators.
9 * Templated solvers for equations up to degree 4.
10 * Simple cross-platform logging system.
11 * Macros for identifying platforms as well as convenience operations such as unused parameters and debug code.
12 * A random number generator based on the new C++11 random library.
13 * Wrappers for shaders and textures.
14 * and more...
15 
16 ## Dependencies
17 For the most part, Atlas is a self-contained library. It does have two dependencies that are **not** included with Atlas. These are:
18 
19 * GLEW. Used only in Linux and Windows. This is required for Atlas.
20 * CMake. Cross-platform build system employed by Atlas. This is required for all platforms.
21 
22 Internally, Atlas ships with the following dependencies:
23 
24 * GLFW. Used for window and context creation.
25 * GLM. Used for all math, vector, matrix, and quaternion operations.
26 * GLI. Used for loading textures.

28 ## Quick Start with Atlas
29 The easiest way to use Atlas is by using CMake. In your project's CMakeLists file, add the following lines:
30 
31 ~~~{.CMake}
32 add_subdirectory(${ATLAS_DIR})
33 ...
34 include_directories(${ATLAS_INCLUDE_DIRS})
35 ...
36 target_link_libraries(${APP_NAME} ${ATLAS_LIBRARIES})
37 ~~~

55 ## Future Plans, Features, etc.
56 The following are a few minor features that are planned for Atlas and that will be slowly added (in no particular order):
57 
58 * Increased support for texture files. In particular, expansion to PPM, BMP, and PNG is planned.
59 * Wrapper for OpenGL framebuffer objects.
60 * Support for logging to a file.
61 * more to come...
